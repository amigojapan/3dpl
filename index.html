An educational real-time interpreted language based on Unity3d's JS for new programmers to learn programming and 3D game programming at the same time, and while having fun! It includes 18 tutorials, which include how to make a simple cube how to color the cubes how to make 3D brakeout(arkanoid) and how to make a 3D space first person shooter and how to make a 3D fist person Platform game. Also you can upload your programs to the (cloud server, still in development stage) and enjoy a huge library of programs made by other users!(when this feature becomes available)

A word of advice to novice programmers
Many of us seasoned programmers started with BASIC and LOGO, simple programming languages where you could type a few commands and get results. it was fun. The debate today between programmers is "What programming language should I learn first?", how to answer this question is a matter of opinion. The answers range from Start with C++(a very difficult language) to python(a rather easy language). Ofcourse, programming comes easier to some than others. Usually I would recommed starting with python for a regular person who can easily grasp difficult concepts quickly. But I have come across some people that really can't seem to grasp the very basics of programming. for such people, I recommend the following: First learn scratch , which is a programming language made by MIT(you can now use scratch in a flash application(it is still in beta testing)), and it is so simple that even kids can enjoy it. then learn my 3D programming language. It has more advanced concepts than scratch (like C style loops and functions, and classes) but it is still simple to learn and fun. Then learn python. it is a general purpose language that will allow you to do many tasks, I recommend the pygame library once you have finished the python tutorial(this will allow you to make fun games in python). After this, you should be able to learn any other programming langauge out there.
If you find that any of these languages seem too easy for you, then skip to the next one, but try staring from scratch, then 3dpl, then python then whatever language you want.
Also it is very important that you self study. it is never enough to just study what your school tells you to study!
